# tiny dinos on-chain renderer — contracts

Foundry project for the fully on-chain renderer. See [`../ONCHAIN.md`](../ONCHAIN.md)
for the full architecture.

## Setup

```bash
forge install foundry-rs/forge-std   # one-time; provides forge-std/
```

The blobs the contracts read (`../build/out/*.bin`) and the test fixtures
(`test/fixtures/*`) are generated by the Python build:

```bash
python3 ../build/build.py
```

## Test

```bash
forge test                                  # sample (all 5 orders, 15 uniques,
                                            # alpha tokens) + per-chain checks
forge test --match-test test_AllTokens -vv  # full 10,000 tokens via keccak hashes
```

The differential test (`test/DinoRenderer.t.sol`) deploys `DinoStorage` +
`DinoRenderer`, loads the real blobs, and asserts `imageSVG` / `metadataJSON` /
`tokenURI` equal the Python-generated fixtures (proven exact vs the original
collection).

## Layout

```
src/
  DinoStorage.sol   SSTORE2 blob holder (sprites, offsets, token records)
  DinoRenderer.sol  compositing + SVG + metadata JSON; tokenURI / metadataJSON / imageSVG
  DinoData.sol      AUTOGENERATED constants (do not edit) — run gen_solidity.py
  lib/SSTORE2.sol   minimal SSTORE2
  lib/Base64.sol    standard base64 encoder
script/Deploy.s.sol per-chain deploy + blob loading
test/DinoRenderer.t.sol  differential test
```

`DinoData.sol` is regenerated from `build/out/manifest.json`; never hand-edit it.
